=== HIGH-LEVEL ARCHITECTURE ===

Core flow:

strategy config
-> scanner creates signal
-> preview creates order_intent
-> optional auto-submit sends Alpaca paper order
-> broker reconciliation imports orders/fills/positions
-> exit cycle evaluates positions
-> post-market maintenance populates trade_cases
-> future AI review reads trade_cases and proposes strategy improvements

Execution rule:
- All trade execution must flow through previewed order_intents.
- Alpaca is source of truth for broker orders, fills, and positions.
- AI can recommend strategy changes only; it must not auto-change live logic.

   - Current cron:

   - During EDT this runs every 5 minutes from 10:00am through 3:55pm Eastern.
   - Render cron is UTC-only and not DST-aware. When Eastern switches to EST, change to:
     */5 15-20 * * 1-5
     for 10:00am through 3:55pm EST.

2. stocks-api-market-exits
   - Exit-only cycle: reconcile + exit evaluation + optional exit submit.
   - Endpoint:
     /api/v1/jobs/market-cycle-exits?limit=100&order_limit=100&fill_page_size=100&phase_timeout_seconds=45
   - Current cron:
     */1 13-20 * * 1-5
   - During EDT this runs every minute from about 9:00am through 4:59pm Eastern.
   - This remains separate from entry; entries stop before 4pm, exits can keep managing positions.

3. stocks-api-market-maintenance
   - Pre/post-market maintenance.
   - Endpoint:
     /api/v1/jobs/market-maintenance?phase=auto&news_enabled=false
   - Current cron:
     30 12,21 * * 1-5
   - Post-market maintenance also populates trade_cases.

Emergency stops:
- TRADING_AUTOMATION_ENABLED=false stops all auto-submit.
- MARKET_CYCLE_SUBMIT_ENABLED=false stops submit while keeping scans/reconcile.
- SCHEDULED_JOBS_ENABLED=false stops cron runner execution.
- MARKET_CYCLE_EXIT_ENABLED=false pauses exit evaluation.
